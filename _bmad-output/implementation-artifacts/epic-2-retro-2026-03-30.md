## 2. O que Foi Bem

### Velocidade da Implementação
A arquitetura FastAPI assíncrono + React com lazy loading resultou em uma interface perceptivelmente rápida com poucos gargalos — validado por Glauber Costa após uso real.

### Cobertura de Testes
O fluxo TDD com fixtures de isolamento multi-tenant (`tenant_a`, `tenant_b`) garantiu 100% dos critérios de aceitação cobertos antes de integrar ao ambiente real.

### Segurança por Design
- `jti` (JWT ID) único por token: previne colisão de hash
- `refresh_token` em cookie `httpOnly`: resistente a XSS
- Hash SHA-256 do token no banco: nunca armazena token bruto
- Reuse detection com revogação em cascata: resposta imediata a sessão comprometida

### TenantScope como Padrão Reutilizável
A classe `TenantScope` estabeleceu um padrão claro e seguro para todas as queries de negócio do Epic 3 em diante. Isolamento multi-tenant documentado, testado e pronto para uso.

### AuthContext com Silent Refresh
O padrão de token em memória + silent refresh ao montar + interceptor de 401 entregou uma experiência de sessão robusta sem depender de `localStorage`.

## 3. Desafios e Raízes

### 3.1 Circular Import — `base.py` ↔ `models.py`
**Impacto:** API não inicializava em produção (container crashava).
**Causa raiz:** `RefreshToken` foi importado em `base.py` por conveniência, violando a regra unidirecional do SQLAlchemy.
**Solução:** Mover importações de models para `conftest.py` e `alembic/env.py`.
**Preocupação do project lead:** Glauber sinalizou este ponto como o mais preocupante do epic, dado o nível de expertise esperado do time.

### 3.2 Migration Alembic Não Aplicada em Produção
**Impacto:** Endpoint `/register` retornava 500 após criar `RefreshToken` model.
**Causa raiz:** Testes usam `Base.metadata.create_all()` (automático). PostgreSQL depende de migrations explícitas.
**Solução:** `alembic revision --autogenerate` + `alembic upgrade head`.

### 3.3 CI Não Valida Build Docker
**Impacto:** Erros de import só aparecem quando o container tenta inicializar.
**Causa raiz:** O `ci.yml` executa pytest mas não faz `docker build` + boot real da API.
**Action item para Epic 3:** Ver seção 5.

### 3.4 Datetime Naive vs Aware (SQLite/PostgreSQL)
**Impacto:** 4 testes de refresh falhando com `TypeError: can't compare offset-naive and offset-aware datetimes`.
**Causa raiz:** SQLite retorna datetimes sem timezone; PostgreSQL retorna com timezone. A diferença de drivers não estava documentada.
**Solução:** Usar `.replace(tzinfo=None)` consistentemente.
